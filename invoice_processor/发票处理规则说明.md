# 发票智能筛选系统 - 处理规则说明

## 一、系统概述

发票智能筛选系统用于从发票池中自动筛选、验证和分类特定月份的可报销发票。

### 主要功能
- 发票信息OCR识别
- 发票合规性验证
- 智能选择最优发票组合
- 自动分类和文件管理
- 生成报销报告

## 三、发票验证规则

### 1. 发票类型识别
- 根据发票"货物或应税劳务、服务名称"字段识别类型
- 匹配类型关键词列表
- 排除无效模式（如餐饮类排除充值权益）

### 2. 开票日期验证
- 发票必须在 **3个月账期内**
- 账期以报销月份为准（如报销2026-01，则接受2025-11至2026-01的发票）

## 四、智能选择策略

### 选择优先级

1. **凑够限额**：优先选择总金额最接近补贴上限的组合
2. **留足后路**：在金额相近时，优先选择：
   - 月份较早的发票
   - 金额较小的发票

### 选择结果分类

| 分类 | 说明 | 文件处理 |
|------|------|----------|
| **选中** | 符合规则且被选中用于报销 | 移动到 `output/{month}/` |
| **多余** | 符合规则但金额超需 | 保留在 `invoice/`，重命名为 `[多余]原文件名` |
| **错误** | 不符合规则（重复、超期、抬头错误等） | 移动到 `output/errors/` |

### 错误类型及标记

| 错误代码 | 说明 | 文件名格式 |
|----------|------|------------|
| 超期 | 超过3个月账期 | `[错误-超期]原文件名` |
| 抬头错误 | 购方名称或税号不匹配 | `[错误-抬头错误]原文件名` |
| 黑名单 | 销方在黑名单中 | `[错误-黑名单]原文件名` |
| 未知类型 | 无法识别发票类型 | `[错误-识别失败]原文件名` |
| 重复 | 发票号码重复 | `[错误-重复]原文件名` |

## 五、文件命名规则

### 选中发票
```
{类型}_{金额}元_{销方名称}_{唯一标识}.pdf
```
示例：`餐饮_108.91元_武汉市渔焰餐饮有限公司_470541.pdf`

### 多余发票
```
[多余]{类型}_{金额}元_{销方名称}_{唯一标识}.pdf
```
示例：`[多余]餐饮_65.35元_武汉东湖新技术开发区仓桥婷婷餐饮店_902081.pdf`

### 错误发票
```
[错误-{原因}]{类型}_{金额}元_{销方名称}_{唯一标识}.pdf
```
示例：`[错误-黑名单]餐饮_50.00元_某某餐饮店_123456.pdf`

### 重复发票
```
[错误-重复]原文件名.pdf
```

## 六、备份规则

### 增量备份
- 备份位置：`invoice_bak/`
- 备份策略：**只复制新增文件**，不覆盖已存在的备份
- 备份时机：每次处理前自动执行

---

## 七、使用方法

### 基本命令
```bash
python3 main.py --month 2026-01
```

### 完整参数
```bash
python3 main.py --month 2026-01 --invoice-pool ../invoice --output-dir ../output
```

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--month` | 报销月份（YYYY-MM） | 必填 |
| `--invoice-pool` | 发票池目录 | `invoice` |
| `--output-dir` | 输出目录 | `output` |

## 八、输出结果

### 目录结构
```
output/
├── 2026-01/              # 可报销发票
├── errors/               # 错误发票
└── 报销报告_2026-01.md   # 报销报告

invoice/                  # 发票池（保留多余发票）
invoice_bak/              # 备份目录
```

### 报告内容
- 汇总信息（总数、可报销数、多余数、错误数）
- 金额达标情况（对比补贴上限）
- 可报销发票明细
- 多余发票明细
- 错误发票明细

---

## 九、处理流程

```
1. 备份发票池（增量备份）
   ↓
2. 扫描发票池PDF文件
   ↓
3. OCR识别发票信息
   ↓
4. 发票号码重复检测
   ↓
5. 合规性验证（类型、日期、抬头、黑名单）
   ↓
6. 智能选择最优组合
   ↓
7. 文件处理（移动/重命名）
   ↓
8. 生成报销报告
```
